Welcome to the Central Data API reference documentation. This GraphQL API provides comprehensive access to GRID's static esports data, including teams, players, tournaments, series, and content catalogs.

## Quick Start

The Central Data API is a GraphQL endpoint that allows you to query and mutate static esports data.

### Endpoint URLs

Choose the endpoint based on your access level:

**Open Access:**
```
https://api-op.grid.gg/central-data/graphql
```

**Commercial & Competitor Access:**
```
https://api.grid.gg/central-data/graphql
```

### Authentication

Include your API key in the request header:

```bash
x-api-key: YOUR_API_KEY
```

### Example Query

```graphql
query GetTournament {
  tournament(id: "tournament-id") {
    id
    title {
      name
      abbreviation
    }
    startDate
    endDate
  }
}
```

## Key Concepts

### GraphQL Schema

The Central Data API follows GraphQL best practices and conventions:

- **Cursor-based pagination** for list queries using the Relay connection pattern
- **Strongly typed schema** ensuring data consistency and validation
- **Flexible filtering** allowing complex queries with multiple filter conditions
- **Nested queries** enabling efficient data retrieval in a single request

### Entity Relationships

The API models relationships between esports entities:

- **Tournaments** contain multiple **Series**
- **Series** involve **Teams** and **Players**
- **Teams** have multiple **Players** across different **Titles**
- **Content Catalogs** define game-specific entities like characters, maps, and items
